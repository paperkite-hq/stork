# Keyboard Shortcuts

Stork is keyboard-first. Most actions have a shortcut, and anything you can do with the mouse you can do without taking your hands off the keyboard.

Press <kbd>?</kbd> anywhere in the app to see this list as a popup.

## Navigation

| Shortcut | Action |
|---|---|
| <kbd>j</kbd> or <kbd>↓</kbd> | Next message |
| <kbd>k</kbd> or <kbd>↑</kbd> | Previous message |
| <kbd>Enter</kbd> | Open the focused message |
| <kbd>Escape</kbd> | Close the current view / go back |

## Message actions

| Shortcut | Action |
|---|---|
| <kbd>r</kbd> | Reply |
| <kbd>a</kbd> | Reply all |
| <kbd>f</kbd> | Forward |
| <kbd>s</kbd> | Star / unstar |
| <kbd>u</kbd> | Toggle read / unread |
| <kbd>d</kbd> | Delete |
| <kbd>e</kbd> | Archive (removes the Inbox label; message stays in All Mail) |
| <kbd>x</kbd> | Select / deselect (for multi-message actions) |

## Compose & search

| Shortcut | Action |
|---|---|
| <kbd>c</kbd> | Compose new message |
| <kbd>⌘</kbd> + <kbd>Enter</kbd> / <kbd>Ctrl</kbd> + <kbd>Enter</kbd> | Send the message you're composing |
| <kbd>/</kbd> | Focus the search box |

## Help

| Shortcut | Action |
|---|---|
| <kbd>?</kbd> | Show this cheatsheet |

## Notes

- Shortcuts are ignored while you're typing in an input or textarea — so <kbd>r</kbd> in the subject line types an "r", it doesn't trigger Reply. The exception is <kbd>⌘</kbd>/<kbd>Ctrl</kbd> + <kbd>Enter</kbd> inside the compose form, which always sends.
- On macOS, <kbd>⌘</kbd> is the Command key. On Windows and Linux, substitute <kbd>Ctrl</kbd>.
- These bindings are inspired by Gmail's keyboard shortcuts, so if you're coming from Gmail most of them should already be in your muscle memory.
